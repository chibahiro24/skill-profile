# キャリアパス推奨機能ガイド

**最終更新**: 2026-02-13  
**対象ユーザー**: 全職種（エンジニア、営業、マーケティング、人事、経営層等）

---

## 📖 概要

Skill Profileのキャリアパス推奨機能は、**あなたと似た経歴・スキルを持つ人々が辿ったキャリアパス**を分析し、実現可能なキャリア選択肢を提示します。

### 🎯 この機能でできること

1. **類似プロファイルの発見**: あなたと似たスキル・経験を持つ人を見つける
2. **キャリアパスの可視化**: 彼らがどのようなキャリアを歩んだかを確認
3. **スキルギャップ分析**: 目標に必要なスキルと現状の差を把握
4. **具体的アクション推奨**: 何を学び、何を実践すべきかを提案
5. **透明性のある推奨**: なぜその推奨なのかを完全に説明

---

## 🚀 クイックスタート

### ステップ1: プロフィールの充実

キャリアパス推奨の精度を高めるために、以下を登録してください：

- ✅ 現在のポジション・役職
- ✅ 主要なスキル（ソフトスキル + ハードスキル）
- ✅ 経験年数
- ✅ 業界・職種
- ✅ これまでのキャリア履歴

### ステップ2: 類似プロファイルを発見

```
ダッシュボード → 「キャリアパス分析」 → 「類似プロファイルを発見」
```

システムが自動的に、あなたと似たスキル・経験を持つユーザーを検索します。

**表示される情報:**
- 類似度スコア（0-100）
- なぜ類似と判定されたか（詳細説明）
- 共通するスキル
- 経験年数の比較

### ステップ3: キャリアパスを探索

類似ユーザーが辿ったキャリアパスが、インタラクティブなグラフで表示されます。

**例:**
```
現在: プロジェクトマネージャー
  ↓ 3年（47人が遷移）
シニアプロジェクトマネージャー
  ↓ 2年（32人が遷移）
プロダクトマネージャー
  ↓ 4年（18人が遷移）
事業部長
```

### ステップ4: 目標を設定

興味のあるポジションを目標として設定します。

```
「プロダクトマネージャー」を目標に設定
```

### ステップ5: スキルギャップを確認

目標に必要なスキルと、現在のスキルの差が表示されます。

**例:**
```
必要なスキル:
- プロダクト戦略: 85点 → 現在: 65点（ギャップ: 20点）★★★★★
- データ分析: 80点 → 現在: 50点（ギャップ: 30点）★★★★★
- 財務理解: 70点 → 現在: 45点（ギャップ: 25点）★★★★☆
```

### ステップ6: 推奨アクションを実行

システムが具体的なアクションプランを提示します。

## 💡 詳細機能

### 1. 類似プロファイル検索

#### 類似度の計算方法

システムは以下の要素を考慮して類似度を計算します：

| 要素 | 重み | 説明 |
|------|------|------|
| ソフトスキル | 40% | コミュニケーション、時間管理、問題解決等 |
| リーダーシップ | 15% | チーム運営、意思決定、影響力等 |
| ビジネススキル | 20% | 戦略思考、顧客志向、財務理解等 |
| アントレプレナーシップ | 10% | 構想力、実行力、リスクテイク等 |
| 経営スキル | 5% | 組織マネジメント、資金管理等 |
| ハードスキル | 10% | 職種別の専門スキル |

**重み付けはカスタマイズ可能です！**

設定画面で、あなたが重視したい要素の重みを調整できます。

#### 推奨理由の表示

「なぜ類似と判定されたか」を明確に表示します。

**例:**
```
山田太郎さんとの類似度: 87点

主な類似理由:
1. プロジェクトマネジメント: あなた 90点 vs 相手 88点（寄与度 25%）
2. コミュニケーション: あなた 85点 vs 相手 87点（寄与度 22%）
3. 問題解決能力: あなた 82点 vs 相手 80点（寄与度 18%）
4. 経験年数: あなた 7年 vs 相手 8年（寄与度 15%）
5. 業界: 同じIT業界（寄与度 12%）

計算プロセスを見る →
```

「計算プロセスを見る」をクリックすると、類似度の計算式とパラメータが表示されます。

### 2. キャリアパス分析

#### キャリアグラフの見方

```
┌─────────────────────────────────────────────┐
│  現在のポジション: プロジェクトマネージャー      │
└─────────────────────────────────────────────┘
              │
              ├─→ シニアPM（遷移確率 65%、平均3年）
              │        │
              │        ├─→ プロダクトマネージャー（遷移確率 45%、平均2年）
              │        │        │
              │        │        └─→ 事業部長（遷移確率 30%、平均4年）
              │        │
              │        └─→ プログラムマネージャー（遷移確率 30%、平均3年）
              │
              └─→ エンジニアリングマネージャー（遷移確率 25%、平均2年）
                       │
                       └─→ VP of Engineering（遷移確率 15%、平均5年）
```

#### 実現可能性スコア

各キャリアパスには「実現可能性スコア」が表示されます。

**計算式:**
```
実現可能性 = (遷移確率 × 60%) + (スキルマッチ度 × 40%)
```

**例:**
```
パス: PM → シニアPM → プロダクトマネージャー
実現可能性: 78点

内訳:
- 遷移確率: 65% × 45% = 29% → 正規化して75点
- スキルマッチ度: 82点
- 総合: 75 × 0.6 + 82 × 0.4 = 78点

このパスを辿った人: 47人
平均到達期間: 5.2年
成功率: 89%（47人中42人が目標達成）
```

### 3. スキルギャップ分析

#### ギャップの可視化

目標ポジションに必要なスキルと、現在のスキルの差が視覚的に表示されます。

**レーダーチャート:**
```
        プロダクト戦略
              /\
             /  \
  データ分析 /    \ 顧客理解
           /      \
          /        \
         /__________\
   財務理解        技術理解

青線: 目標レベル
赤線: 現在のレベル
```

#### 優先順位付け

システムが自動的に、習得すべきスキルの優先順位を計算します。

**優先順位の計算式:**
```
優先度 = (ギャップの大きさ × スキルの重要度) / 習得難易度
```

**例:**
```
優先順位 1: データ分析 ★★★★★
  現在: 50点 → 目標: 80点（ギャップ: 30点）
  重要度: 95/100
  習得難易度: 6/10
  優先度スコア: 4.75
  理由: プロダクトマネージャーに最も必要なスキル

優先順位 2: 財務理解 ★★★★☆
  現在: 45点 → 目標: 70点（ギャップ: 25点）
  重要度: 85/100
  習得難易度: 7/10
  優先度スコア: 3.04
  理由: 事業判断に不可欠
```

### 4. 具体的アクション推奨

#### 学習リソース推奨

スキルギャップを埋めるための最適な学習リソースを推奨します。

**推奨の根拠:**
- このリソースを使った人の平均スキル向上度
- あなたの現在のレベルとの適合性
- 学習期間・コスト

**例:**
```
データ分析スキルを習得するために:

1. ★★★★★ 「実践データ分析入門」（Udemy）
   期間: 3ヶ月、費用: 15,000円
   期待スキル向上: 50点 → 72点（+22点）
   
   推奨理由:
   - 類似ユーザー127人が受講
   - 平均向上度: +23点（最も高い）
   - あなたのレベル（50点）に最適
   - 修了率: 87%（高い）
   
   このコースを修了した人の声:
   「実務で使えるスキルが身についた」（★★★★★）
   「初心者でも分かりやすい」（★★★★☆）

2. ★★★★☆ 「データドリブン意思決定」（書籍）
   期間: 1ヶ月、費用: 3,000円
   期待スキル向上: 50点 → 65点（+15点）
   
3. ★★★☆☆ 社内プロジェクトへの参加
   推奨プロジェクト: 「新規サービスのKPI設計」
   期間: 3ヶ月
   期待スキル向上: 50点 → 68点（+18点）
```

#### プロジェクト・実践機会の推奨

学習だけでなく、実践機会も推奨します。

**例:**
```
実践機会:

1. 社内プロジェクト「新規サービスのKPI設計」
   スキル向上: データ分析 +18点、プロダクト戦略 +12点
   期間: 3ヶ月
   難易度: 中
   
   推奨理由:
   - あなたのスキルセットに適合
   - プロダクトマネージャーへの典型的な経験
   - 類似ユーザー43人がこの種のプロジェクトを経験

2. オープンソースプロジェクトへの貢献
   推奨プロジェクト: [プロジェクト名]
   スキル向上: 技術理解 +15点
```

#### メンター・コミュニティ推奨

目標達成を支援してくれる人やコミュニティを推奨します。

**例:**
```
メンター候補:

佐藤花子さん（プロダクトマネージャー、経験10年）
- あなたと類似した経歴（類似度: 82点）
- PM → シニアPM → プロダクトマネージャー と遷移
- メンター経験あり（5人を指導）
- 得意分野: データ分析、プロダクト戦略

コミュニティ:

1. Product Manager Japan
   メンバー数: 3,500人
   活動: 月次勉強会、Slack交流
   
2. データ分析勉強会
   メンバー数: 1,200人
   活動: 週次オンライン勉強会
```

### 5. 推奨理由の完全説明

すべての推奨について、「なぜその推奨なのか」を詳細に説明します。

#### 説明の階層

1. **サマリー**: 一文での説明
2. **詳細理由**: データに基づく説明
3. **計算プロセス**: 使用したアルゴリズムとパラメータ
4. **ソースコード**: GitHubのコードへのリンク

**例:**
```
推奨: 「データ分析スキルの習得」

【サマリー】
プロダクトマネージャーに最も必要なスキルで、
現在のギャップが30点と大きいため、優先度が最も高いです。

【詳細理由】
1. ギャップの大きさ: 30点（目標80点 - 現在50点）
2. スキルの重要度: 95/100
   - プロダクトマネージャーの95%がこのスキルを持つ
   - 平均スコア: 82点
3. 習得難易度: 6/10（中程度）
4. 優先度スコア: (30 × 0.95) / 6 = 4.75（最高）

【計算プロセス】
使用アルゴリズム: スキルギャップ優先順位付け
パラメータ:
- ギャップ重み: 1.0
- 重要度重み: 0.95
- 難易度補正: 線形（1-10スケール）

【ソースコード】
https://github.com/YOUR_ORG/skill-profile/blob/main/src/algorithms/skill_gap.py#L45-L67
```

## ⚙️ 設定・カスタマイズ

### 類似度の重み付け変更

デフォルトの重み付けを自分好みに調整できます。

```
設定 → キャリアパス設定 → 類似度の重み付け
```

**例:**
```
ソフトスキル: 40% → 50%（重視したい）
ハードスキル: 10% → 5%（あまり重視しない）
```

### 推奨の頻度設定

キャリアパス推奨の通知頻度を設定できます。

```
設定 → 通知設定 → キャリアパス推奨

- 毎週（デフォルト）
- 隔週
- 月次
- 通知しない
```

### データ利用範囲の制御

推奨に使用されるデータの範囲を制御できます。

```
設定 → プライバシー設定 → キャリアパス推奨

□ 類似プロファイル検索に自分のデータを使用することを許可
□ キャリアパス分析の統計に自分のデータを含める
□ 学習リソースの効果測定に自分のデータを使用
```

## 📞 サポート

### よくある質問

**Q: 類似プロファイルが見つからない場合は？**

A: プロフィール情報が不足している可能性があります。以下を確認してください：
- 主要スキルが5つ以上登録されているか
- キャリア履歴が登録されているか
- 業界・職種が設定されているか

**Q: 推奨理由が理解できない場合は？**

A: 「詳細を見る」→「計算プロセスを見る」で、アルゴリズムの詳細を確認できます。
それでも分からない場合は、[GitHub Discussions](https://github.com/YOUR_ORG/skill-profile/discussions)で質問してください。

**Q: 推奨が不適切だと感じた場合は？**

A: フィードバックボタンから報告してください。コミュニティでレビューされ、アルゴリズム改善に活用されます。
